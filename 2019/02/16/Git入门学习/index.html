Git入门学习-介绍与安装

Posted on 2019-02-16 | In Git | Comments:

学习Git的主要途径是git的官网,有一本书叫,在官网上可以免费下载,而且有对应的中文版本。

了解Git

首先Git是一开始由Linus写的，可能是因为Bitkeeper不再免费授权之后，Linus便觉得停止Linux的开发，知道新的工具被开发出来，管理Linux。后来Linus花了两周就写出了git，这里有一个Linus在GoogleTalk介绍Git的视频。

版本控制

版本控制是一种记录一个或若干文件内容变化，以便将来查阅特定版本修订情况的系统。 在本书所展示的例子中，我们对保存着软件源代码的文件作版本控制，但实际上，你可以对任何类型的文件进行版本控制。比如二进制文件。最暴力的办法就是复制整个文件然后压缩，备注版本即可。但这种方式简单但容易犯错，因此大佬们便开发软件来控制这一问题。

本地版本管理系统


采用简单数据库来记录文件的历次更新差异。

集中化的版本控制


顾名思义，采用集中管理的服务器，保存所有文件的修订版本。

分布式版本控制系统


分布式版本控制系统（Distributed Version Control System，简称 DVCS）面世了。 在这类系统中，像 Git、Mercurial、Bazaar 以及 Darcs 等，客户端并不只提取最新版本的文件快照，而是把代码仓库完整地镜像下来。 这么一来，任何一处协同工作用的服务器发生故障，事后都可以用任何一个镜像出来的本地仓库恢复。 因为每一次的克隆操作，实际上都是一次对代码仓库的完整备份。
